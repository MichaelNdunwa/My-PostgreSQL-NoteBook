# Chapter 3: Aggregations & Grouping

Aggregation functions compute a single result value from a set of input values. In SQL, they are extremely powerful when combined with the `GROUP BY` clause.

---

## 1. Standard Aggregation Functions

PostgreSQL provides several standard aggregate functions:
- `COUNT()`: Counts the number of rows.
- `SUM()`: Calculates the total of numeric columns.
- `AVG()`: Calculates the mathematical average.
- `MIN()`: Finds the smallest value.
- `MAX()`: Finds the largest value.

```sql
SELECT 
    COUNT(*) AS total_products,
    SUM(stock) AS total_inventory,
    AVG(price) AS average_price,
    MIN(price) AS cheapest_product,
    MAX(price) AS most_expensive_product
FROM products;
```

---

## 2. Grouping Rows with `GROUP BY`

The `GROUP BY` clause divides the rows of a table into groups. For each group, you can apply aggregate functions:

```sql
SELECT 
    category_id, 
    COUNT(*) AS product_count,
    AVG(price) AS average_price
FROM products
GROUP BY category_id;
```

Let's join with the `categories` table to show the category name instead of just the ID:
```sql
SELECT 
    c.name AS category_name,
    COUNT(p.id) AS product_count,
    ROUND(AVG(p.price), 2) AS average_price
FROM products p
INNER JOIN categories c ON p.category_id = c.id
GROUP BY c.name
ORDER BY product_count DESC;
```

---

## 3. Filtering Groups with `HAVING`

The `WHERE` clause filters rows *before* groups are created. To filter groups *after* they are created and aggregated, use the `HAVING` clause:

```sql
-- Find categories with an average product price greater than $100
SELECT 
    c.name AS category_name,
    COUNT(p.id) AS product_count,
    ROUND(AVG(p.price), 2) AS average_price
FROM products p
INNER JOIN categories c ON p.category_id = c.id
GROUP BY c.name
HAVING AVG(p.price) > 100.00;
```

---

## 4. Useful PostgreSQL-Specific Aggregations

PostgreSQL offers additional advanced aggregation functions, such as string aggregation and array aggregation.

### String Aggregations (`string_agg`)
Concatenates values into a single string, separated by a delimiter:
```sql
SELECT 
    role, 
    string_agg(name, ', ') AS users_in_role
FROM users
GROUP BY role;
```

### Array Aggregations (`array_agg`)
Collects values into a PostgreSQL array:
```sql
SELECT 
    c.name AS category_name,
    array_agg(p.name) AS product_list
FROM products p
INNER JOIN categories c ON p.category_id = c.id
GROUP BY c.name;
```
