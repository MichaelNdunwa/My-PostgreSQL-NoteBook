# Chapter 7: Performance Tuning & Indexing

Writing query logic that returns the correct results is the first step. Ensuring that the queries execute efficiently at scale is the second. PostgreSQL provides powerful analysis tools and indexing systems to make your database run at lightning speed.

---

## 1. Analyzing Queries with `EXPLAIN`

The `EXPLAIN` statement shows the execution plan generated by the PostgreSQL query planner for a given query.

- `EXPLAIN`: Shows the estimated cost plan without actually executing the query.
- `EXPLAIN ANALYZE`: Executes the query and displays the actual execution times and resource plans.

> [!WARNING]
> Since `EXPLAIN ANALYZE` actually runs the query, be careful when using it with queries that modify data (like `INSERT`, `UPDATE`, or `DELETE`). You may want to wrap them in a transaction that gets rolled back.

```sql
EXPLAIN ANALYZE
SELECT * FROM users WHERE email = 'alice@example.com';
```

In the output, you will see terms like:
- **Seq Scan (Sequential Scan)**: The database scans the entire table row-by-row. Slow for large tables.
- **Index Scan**: The database uses an index to locate the matching rows directly. Extremely fast.
- **Planning Time**: Time taken to generate the execution plan.
- **Execution Time**: Actual execution time of the query.

## 2. PostgreSQL Index Types

Indexes are lookup structures that allow the database to search rows faster. PostgreSQL supports several index types:

### B-Tree Indexes (Default)
Suitable for most queries, sorting, and range lookups (`=`, `<`, `<=`, `>`, `>=`, `BETWEEN`, `IN`).
```sql
-- Creating an index on user emails (which are frequently queried)
CREATE INDEX idx_users_email ON users(email);
```

After creating this index, run the query again:
```sql
EXPLAIN ANALYZE
SELECT * FROM users WHERE email = 'alice@example.com';
```
You should notice the query planner switch from a `Seq Scan` to an `Index Scan` (or `Index Only Scan`), and the execution time will drop significantly on large datasets.

### GIN Indexes (Generalized Inverted Index)
Ideal for indexing multi-valued columns like JSONB or Arrays. A GIN index allows PostgreSQL to index the internal keys and values of JSONB documents.

```sql
-- Create a GIN index on the specs JSONB column
CREATE INDEX idx_products_specs ON products USING gin(specs);
```

Let's test search using the containment operator after indexing:
```sql
EXPLAIN ANALYZE
SELECT name, specs
FROM products
WHERE specs @> '{"brand": "GearUp"}';
```

---

## 3. Covering Indexes (`INCLUDE`)

You can create an index that includes additional columns that aren't part of the search key, allowing "Index Only Scans" which fetch values entirely from the index without reading the table pages:

```sql
CREATE INDEX idx_products_category_price 
ON products(category_id) 
INCLUDE(price, name);
```
